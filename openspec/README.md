# 📋 OpenSpec - 规划层

> 管"为什么做这个变更" —— 规划层的知识沉淀

## 职责

- 📝 需求背景与设计目标
- 🏗️ 技术方案与选型理由
- 📊 变更影响分析与风险评估
- 📚 知识积累与经验传承
- 🔄 架构决策追踪

## 目录结构

```
openspec/
├── specs/           # 功能/非功能需求规格
├── decisions/       # 架构决策记录 (ADR)
└── changes/        # 变更历史追踪
```

## ADR (架构决策记录)

每个重要技术决策需包含：

1. **Title** - 决策标题
2. **Status** - 提议/接受/废弃/已替代
3. **Context** - 背景与问题描述
4. **Decision** - 决策内容
5. **Consequences** - 决策后果

## 变更记录规范

每次变更需记录：
- 变更原因 (Why)
- 变更内容 (What)
- 影响范围 (Impact)
- 回滚方案 (Rollback)
