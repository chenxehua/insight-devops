# ADR-001: 天鹂可视化运维平台架构选型

**状态**: ✅ 已接受  
**日期**: 2026-05-20  
**决策者**: 平台架构团队

---

## 📋 背景

天鹂可视化运维平台需要支撑大规模可视化运维场景，包括：
- 实时监控大屏渲染
- 指标数据的采集、存储与查询
- 多租户权限管理
- 高可用与弹性扩展

## 🎯 决策

采用 **前后端分离 + 微服务架构**：

| 组件 | 技术选型 | 理由 |
|------|---------|------|
| 前端 | Vue3 + ECharts | 生态成熟，可视化能力强 |
| 后端 | Spring Cloud | 微服务生态完善 |
| 数据库 | MySQL + TimescaleDB | 关系型 + 时序数据 |
| 缓存 | Redis Cluster | 高并发低延迟 |
| 消息队列 | Kafka | 高吞吐日志/事件流 |
| 容器编排 | Kubernetes | 弹性扩展 |

## ✅ 后果

**正面**:
- 组件独立演进，技术栈灵活
- 高可用设计，单点故障不影响全局
- 前后端分离便于并行开发

**负面**:
- 运维复杂度增加
- 服务间通信开销
- 需要完善的监控系统
