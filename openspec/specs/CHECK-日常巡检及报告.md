# 日常巡检及报告

## 1. 概述

### 1.1 功能简介
提供系统日常巡检能力，支持巡检任务配置、自动执行、报告生成和历史对比。

### 1.2 用户故事
作为运维工程师，我希望通过平台定期执行日常巡检，自动生成巡检报告，及时发现潜在风险，减少生产环境故障。

### 1.3 适用范围
适用于日常运维巡检、定期健康检查、巡检报告生成等场景。

### 2.2 功能详情

#### 2.2.1 巡检指标

##### 子功能：指标配置

**业务流程：**
1. 用户进入指标配置页面
2. 添加巡检对象（主机/服务）
3. 配置巡检指标和阈值
4. 保存配置
5. 验证指标可采集

**输入：**
- 巡检对象：主机IP/服务名
- 指标名称：cpu_usage | memory_usage | disk_usage | process_count
- 阈值：warning | critical
- 比较符：> | < | >= | <= | ==

**输出：**
- 指标配置ID
- 配置状态：已保存

**异常处理：**
- 指标不支持：提示"该指标在目标对象上不可用"
- 阈值不合理：提示"阈值设置可能导致频繁告警"

##### 子功能：指标采集

**业务流程：**
1. 巡检任务执行时
2. 系统采集各项指标
3. 与阈值比较
4. 记录比较结果
5. 判定巡检状态

**输入：**
- 巡检对象列表
- 指标列表
- 阈值配置

**输出：**
- 指标采集结果
- 阈值对比结果
- 巡检状态：正常/警告/异常

---

#### 2.2.2 定时配置

##### 子功能：定时任务配置

**业务流程：**
1. 用户创建定时巡检任务
2. 设置执行周期
3. 选择巡检对象和指标
4. 设置执行时间
5. 保存配置

**输入：**
- 任务名称
- 执行周期：每日 | 每周 | 每月 | Cron
- 执行时间：HH:mm
- 巡检对象列表
- 指标列表

**输出：**
- 定时任务ID
- 下次执行时间
- 任务状态

##### 子功能：周期配置

**业务流程：**
1. 用户选择周期类型
2. 配置具体时间
3. 配置每周几/每月几号执行
4. 保存周期配置

**输入：**
- 周期类型：daily | weekly | monthly | cron
- 执行时间详细配置

**输出：**
- 周期配置ID
- Cron表达式（如使用）

---

#### 2.2.3 自动执行

##### 子功能：巡检执行

**业务流程：**
1. 定时任务触发
2. 创建巡检执行实例
3. 按配置采集指标
4. 与阈值比较
5. 记录巡检结果
6. 触发告警（如有异常）

**输入：**
- 定时任务ID
- 触发时间

**输出：**
- 巡检执行ID
- 执行状态：成功/部分成功/失败
- 异常数量
- 巡检耗时

**异常处理：**
- 巡检对象不可达：标记为不可达，继续其他对象
- 指标采集失败：跳过该指标，记录错误

##### 子功能：并发控制

**业务流程：**
1. 多个巡检任务同时触发时
2. 系统进行并发控制
3. 限制同时执行的巡检数
4. 排队等待执行

**输入：**
- 任务列表
- 并发限制数

**输出：**
- 执行队列
- 等待时间

---

#### 2.2.4 报告生成

##### 子功能：报告模板配置

**业务流程：**
1. 用户配置报告模板
2. 设置报告内容（必选项、可选项）
3. 设置报告格式
4. 保存模板

**输入：**
- 模板名称
- 包含内容：巡检摘要 | 异常详情 | 趋势分析
- 格式：PDF | HTML | Markdown

**输出：**
- 报告模板ID
- 模板状态

##### 子功能：报告生成

**业务流程：**
1. 巡检完成后
2. 根据模板生成报告
3. 收集巡检数据
4. 格式化报告内容
5. 保存报告文件
6. 发送报告（如配置）

**输入：**
- 巡检执行ID
- 报告模板ID

**输出：**
- 报告ID
- 报告文件路径
- 报告生成时间

- 报告列表

##### 子功能：报告详情

**业务流程：**
1. 用户点击报告
2. 系统加载报告内容
3. 展示巡检摘要

**输入：**
- 报告ID

**输出：**
- 报告完整内容
- 异常项列表

**业务流程：**
1. 用户点击下载
2. 系统生成目标格式文件

**输入：**
- 报告ID
- 文件格式：PDF | HTML | Markdown

**输出：**
- 下载链接
- 文件大小

#### 2.2.6 历史对比

##### 子功能：趋势查看

**业务流程：**
1. 用户选择巡检对象
2. 选择时间范围
3. 系统获取历史数据
4. 生成趋势图
5. 展示指标变化趋势

**输入：**
- 巡检对象
- 指标名称
- 时间范围：7天 | 30天 | 90天

**输出：**
- 趋势图数据
- 环比变化
- 同比变化

##### 子功能：对比分析

**业务流程：**
1. 用户选择两个时间点
2. 选择要对比的指标
3. 系统对比数据
4. 展示差异
5. 高亮异常变化

**输入：**
- 时间点A
- 时间点B
- 巡检对象
- 指标列表

**输出：**
- 对比结果
- 变化百分比
- 异常标记

#### 2.2.7 异常告警

##### 子功能：告警规则配置

**业务流程：**
1. 用户配置巡检告警规则
2. 设置触发条件（阈值、持续次数）
3. 设置告警级别
4. 设置通知方式
5. 保存配置

**输入：**
- 告警条件：指标 + 阈值 + 持续次数
- 告警级别：warning | critical
- 通知方式：邮件 | 钉钉 | 飞书
- 接收人列表

**输出：**
- 告警规则ID
- 规则状态

##### 子功能：告警触发

**业务流程：**
1. 巡检发现异常
2. 判断是否满足告警条件
3. 触发告警
4. 发送通知
5. 记录告警事件

**输入：**
- 巡检结果
- 告警规则

**输出：**
- 告警事件
- 通知状态

---

#### 2.2.8 自定义脚本

##### 子功能：脚本配置

**业务流程：**
1. 用户编写自定义巡检脚本
2. 上传到平台
3. 配置脚本参数
4. 设置预期输出格式
5. 保存配置

**输入：**
- 脚本内容/脚本ID
- 执行参数
- 超时时间
- 预期输出格式

**输出：**
- 自定义巡检配置ID
- 配置状态

##### 子功能：脚本执行

**业务流程：**
1. 巡检任务执行到自定义脚本
2. 在目标主机执行脚本
3. 收集脚本输出
4. 解析输出与预期对比
5. 记录结果

**输入：**
- 自定义脚本ID
- 执行目标
- 执行参数

**输出：**
- 脚本执行结果
- 输出对比结果

---

**业务流程：**
1. 用户创建巡检模板

**输出：**
- 模板ID
- 模板状态

**业务流程：**
1. 用户创建巡检任务时
2. 选择已有模板
3. 系统填充模板配置
4. 用户调整后保存

**输入：**
- 模板ID
- 巡检对象

**输出：**
- 预填充的任务配置
- 需要调整的项

| 指标 | 要求 | 说明 |
|------|------|------|

| 指标 | 要求 | 说明 |

| 指标 | 要求 | 说明 |

## 4. 约束与依赖

### 4.1 约束
- 巡检时间避免业务高峰期
- 自定义脚本需审核后使用
- 巡检阈值需合理配置

### 4.2 依赖
- 监控模块（指标采集）
- 报告模块
- 消息通知模块
