# 镜像管理

## 1. 概述

### 1.1 功能简介
统一管理 Docker 镜像仓库，支持镜像构建、上传、下载、版本管理和安全扫描。

### 1.2 用户故事
作为运维工程师，我希望通过平台统一管理所有镜像，确保镜像来源可追溯、版本可控，消除环境不一致带来的问题。

### 1.3 适用范围
适用于镜像仓库管理、镜像安全扫描、镜像清理等场景。

### 2.2 功能详情

#### 2.2.1 仓库连接

##### 子功能：仓库配置

**业务流程：**
1. 用户进入仓库管理页面
2. 点击"添加仓库"
3. 选择仓库类型
4. 填写仓库地址和凭证
5. 系统测试连接
6. 保存配置

**输入：**
- 仓库类型：harbor | acr | dockerhub | 自建
- 仓库地址：必填
- 用户名：必填
- 密码/Token：必填
- 仓库名称：必填

**输出：**
- 仓库ID
- 连接状态：成功/失败
- 同步镜像数

**异常处理：**
- 连接失败：提示错误信息（网络/认证错误）
- 凭证过期：提示更新凭证
- 仓库不可达：标记为离线状态

##### 子功能：仓库同步

**业务流程：**
1. 配置完成后
2. 系统同步仓库镜像列表
3. 记录镜像元数据
4. 定期增量同步
5. 手动触发全量同步

**输入：**
- 仓库ID
- 同步类型：增量 | 全量

**输出：**
- 同步状态
- 新增镜像数
- 更新时间

---

#### 2.2.2 镜像传输

##### 子功能：镜像上传

**业务流程：**
1. 用户选择本地镜像
2. 选择目标仓库
3. 标记镜像名称和版本
4. 上传镜像到仓库
5. 显示上传进度
6. 上传完成验证

**输入：**
- 镜像文件：tar包或直接推送
- 目标仓库ID
- 镜像名称：namespace/repo
- 镜像版本：tag

**输出：**
- 上传状态：成功/失败
- 镜像大小
- 仓库地址

**异常处理：**
- 上传中断：支持断点续传
- 网络超时：自动重试3次
- 仓库空间不足：提示清理建议

##### 子功能：镜像下载

**业务流程：**
1. 用户选择要下载的镜像
2. 选择目标服务器
3. 系统从仓库下载镜像
4. 传输到目标服务器
5. 验证镜像完整性

**输入：**
- 镜像仓库ID
- 镜像名称：版本
- 目标服务器列表

**输出：**
- 下载状态
- 下载进度
- 传输耗时

**异常处理：**
- 服务器不可达：跳过并提示
- 下载失败：自动重试

---

#### 2.2.3 版本列表

##### 子功能：版本查看

**业务流程：**
1. 用户进入镜像详情页
2. 展示该镜像的所有版本
3. 显示版本大小、创建时间、推送者
4. 支持版本筛选和搜索
5. 支持版本对比

**输入：**
- 镜像仓库ID
- 镜像名称

**输出：**
- 版本列表
- 版本详情（大小、创建时间、签名状态）

##### 子功能：版本删除

**业务流程：**
1. 用户选择要删除的版本
2. 系统检查版本是否被引用
3. 确认删除
4. 删除镜像文件
5. 更新元数据

**输入：**
- 镜像ID
- 版本列表

**输出：**
- 删除状态：成功/失败
- 释放空间大小

**异常处理：**
- 版本被引用：提示"该版本被以下服务使用"
- 权限不足：提示需要管理员权限

---

#### 2.2.4 镜像构建

##### 子功能：构建配置

**业务流程：**
1. 用户创建镜像构建任务
2. 配置代码仓库和Dockerfile路径
3. 设置构建触发条件
4. 配置构建参数和环境
5. 保存配置

**输入：**
- 代码仓库：Git地址
- Dockerfile路径
- 构建上下文
- 构建参数：ARG、LABLE等
- 触发条件：代码提交 | 手动 | 定时

**输出：**
- 构建配置ID
- 构建规则

##### 子功能：构建执行

**业务流程：**
1. 触发构建（代码提交/手动/定时）
2. 克隆代码仓库
3. 执行构建
4. 推送镜像到仓库
5. 记录构建日志
6. 触发安全扫描（如配置）

**输入：**
- 构建配置ID
- 代码版本：commit ID / branch / tag

**输出：**
- 构建任务ID
- 构建状态：构建中 | 成功 | 失败
- 构建日志
- 镜像版本

**异常处理：**
- 构建失败：记录错误日志，发送告警
- 构建超时（默认1小时）：强制终止
- Dockerfile错误：提示具体错误位置

---

#### 2.2.5 安全扫描

##### 子功能：扫描配置

**业务流程：**
1. 用户配置安全扫描规则
2. 设置扫描触发时机（构建后/手动/定时）
3. 设置漏洞等级阈值
4. 设置阻断规则（如高危漏洞阻断部署）
5. 保存配置

**输入：**
- 扫描触发时机：构建后 | 手动 | 定时 | 从不
- 漏洞等级：严重 | 高危 | 中危 | 低危
- 阻断阈值：阻止部署的最低漏洞等级

**输出：**
- 扫描配置ID
- 配置状态

##### 子功能：扫描执行

**业务流程：**
1. 触发扫描（构建完成/手动/定时）
2. 下载镜像到扫描引擎
3. 执行漏洞扫描
4. 分析扫描结果
5. 生成扫描报告
6. 触发阻断判断

**输入：**
- 镜像ID
- 扫描配置ID

**输出：**
- 扫描任务ID
- 漏洞列表（按等级分类）
- 扫描报告URL
- 阻断状态：阻止 | 通过

**异常处理：**
- 扫描引擎不可用：跳过扫描，记录警告
- 扫描超时（默认30分钟）：标记为扫描超时
- 镜像不存在：提示错误

---

#### 2.2.6 使用统计

##### 子功能：存储统计

**业务流程：**
1. 系统定期统计各仓库存储使用
2. 按镜像/版本分级统计
3. 生成统计报表
4. 展示趋势图

**输入：**
- 仓库ID
- 统计周期

**输出：**
- 总存储使用量
- 各镜像占用排行
- 存储趋势

##### 子功能：使用记录

**业务流程：**
1. 记录镜像的下载和使用情况
2. 统计各版本使用频率
3. 标记未使用镜像
4. 生成清理建议

**输入：**
- 镜像仓库ID
- 未使用时间阈值

**输出：**
- 使用记录列表
- 未使用镜像列表
- 清理建议

---

#### 2.2.7 镜像清理

##### 子功能：清理规则配置

**业务流程：**
1. 用户配置清理规则
2. 设置清理条件（时间、使用状态、大小）
3. 设置保护规则（白名单）
4. 模拟执行查看效果
5. 保存并启用

**输入：**
- 清理条件：最后下载时间 > X天 | 从未被使用 | 大小 > XGB
- 保护规则：镜像名称/版本白名单
- 执行方式：手动 | 自动

**输出：**
- 清理规则ID
- 预计释放空间

##### 子功能：清理执行

**业务流程：**
1. 触发清理任务
2. 按规则筛选镜像
3. 排除保护名单
4. 确认清理列表
5. 执行删除
6. 记录清理日志

**输入：**
- 清理规则ID
- 确认执行

**输出：**
- 清理状态
- 删除镜像数
- 释放空间

**异常处理：**
- 误删：保留删除记录30天内可恢复
- 删除失败：跳过并记录原因

---

#### 2.2.8 镜像签名

##### 子功能：签名配置

**业务流程：**
1. 用户配置镜像签名
2. 生成签名密钥对
3. 配置签名策略（哪些镜像需要签名）
4. 保存配置

**输入：**
- 签名密钥：自动生成或上传
- 签名策略：必签镜像规则

**输出：**
- 签名配置ID
- 公钥信息

##### 子功能：签名与验签

**业务流程：**
1. 镜像构建/上传时
2. 按策略执行签名
3. 将签名附加到镜像
4. 部署时验签
5. 验签失败阻止部署

**输入：**
- 镜像ID
- 签名密钥

**输出：**
- 签名结果：成功/失败
- 签名信息

---

#### 2.2.9 镜像复制

##### 子功能：复制规则配置

**业务流程：**
1. 用户配置镜像复制规则
2. 设置源仓库和目标仓库
3. 设置镜像过滤条件
4. 设置复制策略（手动/自动）
5. 保存配置

**输入：**
- 源仓库ID
- 目标仓库ID
- 镜像过滤：名称/版本正则
- 复制策略：手动 | 同步

**输出：**
- 复制规则ID
- 规则状态

##### 子功能：复制执行

**业务流程：**
1. 触发复制任务
2. 从源仓库拉取镜像
3. 推送到目标仓库
4. 验证复制结果
5. 记录复制日志

**输入：**
- 复制规则ID
- 具体镜像（可选）

**输出：**
- 复制状态：成功/失败
- 复制耗时
- 镜像大小

## 3. 非功能需求

### 3.1 性能需求

| 指标 | 要求 | 说明 |
|------|------|------|
| 镜像上传/下载速度 | ≥10MB/s | - |
| 仓库容量支持 | ≥10TB | - |
| 安全扫描 | 异步执行，不阻断流程 | - |
| 镜像元数据保留 | 1年 | - |

### 3.2 安全需求

| 指标 | 要求 | 说明 |
|------|------|------|
| 凭证安全 | 加密存储 | - |
| 镜像签名 | 支持验签 | - |
| 漏洞报告 | 分级展示 | - |

## 4. 约束与依赖

### 4.1 约束
- 镜像仓库需支持Docker Registry v2协议
- 单个镜像大小不超过10GB
- 清理操作需二次确认

### 4.2 依赖
- Docker Registry
- 镜像安全扫描引擎
- 存储系统

## 5. 风险与问题

| ID | 风险/问题 | 影响 | Mitigation/解决方案 |
|----|-----------|------|---------------------|
| RISK-001 | 镜像泄露 | 高 | 私有仓库权限控制 |
| RISK-002 | 漏洞镜像上线 | 高 | 安全扫描强制卡点 |

## 6. 验收标准

1. 成功连接主流镜像仓库
2. 镜像上传下载功能正常
3. 镜像版本历史可追溯
4. 安全扫描功能正常
5. 镜像清理功能正常
