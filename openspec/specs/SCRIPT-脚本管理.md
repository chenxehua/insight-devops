# 脚本管理

## 1. 概述

### 1.1 功能简介
统一管理运维脚本，提供脚本仓库、版本控制、执行记录和权限控制。

### 1.2 用户故事
作为运维人员，我希望将常用运维脚本集中管理，避免脚本散落在各环境，通过平台统一执行并记录执行日志。

### 1.3 适用范围
适用于日常运维脚本管理、自动化任务配置、批量操作等场景。

### 2.2 功能详情

#### 2.2.1 脚本CRUD

##### 子功能：脚本上传

**业务流程：**
1. 用户进入脚本管理页面
2. 点击"上传脚本"按钮
3. 选择脚本文件或拖拽上传
4. 系统检查文件格式和大小
5. 填写脚本名称、分类、标签
6. 提交保存
7. 系统保存并生成初始版本

**输入：**
- 脚本文件：支持 .sh/.py/.yml/.yaml/.json，最大10MB
- 脚本名称：必填，50字符以内
- 脚本类型：Shell | Python | Ansible | YAML | JSON | Other
- 分类：必填，从预定义分类中选择
- 标签：可选，逗号分隔

**输出：**
- 脚本ID
- 脚本版本
- 上传成功提示

**异常处理：**
- 文件格式不支持：提示"不支持的文件格式"
- 文件过大：提示"文件超过10MB限制"
- 脚本语法错误：提示具体错误位置（不阻断上传）

##### 子功能：脚本编辑

**业务流程：**
1. 用户选择脚本，点击"编辑"
2. 系统展示脚本内容和元数据
3. 用户修改脚本内容
4. 系统实时检查语法
5. 用户填写修改说明
6. 保存新版本

**输入：**
- 脚本ID
- 新脚本内容
- 修改说明：必填，描述本次修改内容

**输出：**
- 新版本号
- 保存成功提示

**异常处理：**
- 语法错误：提示错误，不允许保存
- 并发编辑冲突：提示"脚本已被他人修改，请刷新后重试"

##### 子功能：脚本删除

**业务流程：**
1. 用户选择脚本，点击"删除"
2. 系统显示确认对话框
3. 用户确认删除
4. 系统检查是否有依赖的执行任务
5. 执行软删除（可恢复）或硬删除

**输入：**
- 脚本ID
- 删除类型：软删除（30天后彻底删除）| 硬删除（立即删除）

**输出：**
- 删除状态：成功 | 失败
- 影响的任务数（如有）

**异常处理：**
- 脚本被任务引用：提示"该脚本被X个任务引用，删除后任务将失败"
- 权限不足：提示"只有管理员可以删除脚本"

#### 2.2.2 版本管理

##### 子功能：版本历史查看

**业务流程：**
1. 用户选择脚本，点击"版本历史"
2. 系统展示版本列表
3. 用户选择查看某一版本
4. 系统展示该版本的内容和变更信息

**输入：**
- 脚本ID
- 版本号（可选，默认最新）

**输出：**
- 版本列表（版本号、创建时间、创建人、变更说明）
- 选中版本的内容

**异常处理：**
- 版本不存在：提示"版本不存在"
- 查看超时：提示重试

##### 子功能：版本回滚

**业务流程：**
1. 用户选择历史版本，点击"回滚"
2. 系统显示回滚预览（与当前版本对比）
3. 用户确认回滚
4. 系统创建新版本，内容为旧版本
5. 回滚完成提示

**输入：**
- 脚本ID
- 目标版本号
- 回滚原因：必填

**输出：**
- 新版本号（回滚后的版本）
- 回滚状态：成功

**异常处理：**
- 回滚失败：提示错误原因，保留当前版本
- 回滚权限不足：提示"只有脚本所有者可以回滚"

#### 2.2.3 分类标签

##### 子功能：分类管理

**业务流程：**
1. 管理员进入分类管理页面
2. 查看现有分类列表
3. 新增/编辑/删除分类
4. 分类关联脚本数量统计

**输入：**
- 分类名称：必填，20字符以内
- 分类描述：可选
- 上级分类：可选（支持多级）

**输出：**
- 分类ID
- 分类树结构

**异常处理：**
- 分类名称重复：提示"分类名称已存在"
- 分类下有脚本：不允许删除，提示先迁移脚本

##### 子功能：标签管理

**业务流程：**
1. 用户为脚本添加标签
2. 支持批量添加
3. 支持标签搜索和筛选
4. 标签统计和推荐

**输入：**
- 脚本ID
- 标签列表

**输出：**
- 标签更新结果
- 相关脚本推荐

**异常处理：**
- 标签过长：自动截断
- 标签数量超限（最多10个）：提示"每个脚本最多10个标签"

#### 2.2.4 参数配置

##### 子功能：参数定义

**业务流程：**
1. 用户进入脚本参数配置页面
2. 添加参数定义
3. 设置参数名称、类型、默认值、是否必填
4. 保存参数配置

**输入：**
- 参数名称：必填，字母数字下划线
- 参数类型：string | number | boolean | select | password
- 默认值：可选
- 是否必填：是 | 否
- 参数描述：可选
- 选项列表：select类型必填，逗号分隔

**输出：**
- 参数配置ID
- 参数配置预览

**异常处理：**
- 参数名称不合规：提示"参数名称只能包含字母、数字、下划线"
- 类型与默认值不匹配：提示"默认值与参数类型不匹配"

##### 子功能：参数校验

**业务流程：**
1. 用户执行脚本时填写参数
2. 系统实时校验参数格式
3. 必填项检查
4. 自定义规则校验
5. 校验通过后执行

**输入：**
- 参数名称
- 参数值
- 参数配置

**输出：**
- 校验结果：成功 | 失败
- 错误信息（如有）

**异常处理：**
- 必填项为空：提示"参数XXX为必填项"
- 格式错误：提示"参数XXX格式错误"
- 值不在选项列表中：提示"参数XXX的值不在允许范围内"

#### 2.2.5 脚本执行

##### 子功能：单次执行

**业务流程：**
1. 用户选择脚本，点击"执行"
2. 填写执行参数（如果脚本定义了参数）
3. 选择执行目标（服务器/主机组）
4. 点击"立即执行"
5. 系统创建执行任务
6. 实时推送执行日志
7. 执行完成展示结果

**输入：**
- 脚本ID
- 执行参数：JSON格式
- 执行目标：主机IP或主机组ID
- 执行用户：可选，默认脚本所有者

**输出：**
- 执行任务ID
- 执行状态：执行中 | 成功 | 失败
- 执行耗时
- 执行输出

**异常处理：**
- 目标主机不可达：提示"主机XXX无法连接"
- 执行超时：强制终止，标记为超时
- 执行被取消：停止执行，更新状态

##### 子功能：定时执行

**业务流程：**
1. 用户选择脚本，点击"定时执行"
2. 设置执行时间（一次性或周期）
3. 设置执行参数
4. 设置执行目标
5. 确认创建定时任务
6. 系统在指定时间自动执行

**输入：**
- 脚本ID
- 执行时间：单次时间或Cron表达式
- 执行参数
- 执行目标
- 任务名称：必填

**输出：**
- 定时任务ID
- 下次执行时间
- 任务状态

**异常处理：**
- 时间格式错误：提示"时间格式不正确"
- 定时任务冲突：提示"与现有定时任务冲突"

##### 子功能：周期执行

**业务流程：**
1. 用户设置执行周期（Cron表达式）
2. 设置执行参数和目标
3. 启用任务
4. 系统按周期执行脚本
5. 每次执行记录结果
6. 支持启用/暂停任务

**输入：**
- 脚本ID
- Cron表达式
- 执行参数
- 执行目标
- 周期任务名称

**输出：**
- 周期任务ID
- 执行历史
- 下次执行时间

**异常处理：**
- Cron表达式错误：提示"表达式格式错误"
- 连续执行失败（3次）：自动暂停任务，发送告警

#### 2.2.6 执行日志

##### 子功能：日志记录

**业务流程：**
1. 脚本执行开始
2. 系统创建执行日志记录
3. 实时记录标准输出和标准错误
4. 记录执行状态和耗时
5. 执行结束保存完整日志

**输入：**
- 执行任务ID
- 执行进程ID

**输出：**
- 日志记录ID
- 标准输出
- 标准错误
- 执行状态
- 开始/结束时间

##### 子功能：日志查询

**业务流程：**
1. 用户进入执行日志页面
2. 设置查询条件
3. 执行查询
4. 展示日志列表
5. 点击查看详情
6. 支持日志下载

**输入：**
- 时间范围
- 脚本名称/ID
- 执行状态
- 执行人

**输出：**
- 日志列表
- 总记录数
- 日志内容

**异常处理：**
- 日志已过期：提示"日志保留期限为180天，已过期删除"
- 日志查询超时：提示"查询超时，请缩小时间范围"

---

#### 2.2.7 结果通知

##### 子功能：通知配置

**业务流程：**
1. 用户配置脚本执行通知
2. 选择通知触发条件（成功/失败/超时）
3. 选择通知渠道
4. 设置通知接收人
5. 保存配置

**输入：**
- 脚本ID
- 触发条件：成功 | 失败 | 超时 | 总是
- 通知渠道：邮件 | 短信 | 钉钉 | 飞书
- 接收人列表

**输出：**
- 通知配置ID
- 配置状态：已启用

##### 子功能：通知发送

**业务流程：**
1. 脚本执行完成
2. 系统判断是否满足通知条件
3. 读取通知配置
4. 格式化通知内容
5. 通过指定渠道发送

**输入：**
- 执行任务ID
- 执行结果
- 通知配置ID

**输出：**
- 发送状态：成功 | 失败
- 发送时间

**异常处理：**
- 发送失败：记录失败原因，重试3次
- 渠道配置错误：跳过并记录，发送备用通知

---

#### 2.2.8 依赖管理

##### 子功能：环境依赖配置

**业务流程：**
1. 用户编辑脚本依赖
2. 添加环境依赖（如Python版本、系统包）
3. 设置依赖版本要求
4. 保存配置
5. 执行前检查依赖是否满足

**输入：**
- 脚本ID
- 依赖类型：系统包 | 语言包 | 文件
- 依赖名称
- 版本要求：>= | <= | == | range

**输出：**
- 依赖配置ID
- 依赖检查结果

##### 子功能：执行前检查

**业务流程：**
1. 脚本执行前
2. 读取脚本依赖配置
3. 检查目标主机环境
4. 检查依赖是否满足
5. 如不满足，提示用户

**输入：**
- 脚本ID
- 目标主机

**输出：**
- 检查结果：满足 | 不满足
- 不满足项列表

**异常处理：**
- 依赖不满足：禁止执行，提示安装依赖
- 检查超时：跳过检查，提示可能存在风险

## 3. 非功能需求

### 3.1 性能需求

| 指标 | 要求 | 说明 |
|------|------|------|
| 脚本库容量 | ≥1000个 | - |
| 脚本执行响应 | <5秒 | 任务下发时间 |

### 3.2 安全需求

| 指标 | 要求 | 说明 |
|------|------|------|
| 权限控制 | 脚本操作权限分级 | - |
| 内容审计 | 支持脚本内容审计 | 安全检查 |
| 敏感操作 | 高危脚本执行需审批 | 防止误操作 |

## 4. 约束与依赖

### 4.1 约束
- 脚本编码统一为UTF-8
- 单个脚本文件大小不超过10MB
- 高危脚本（如删除数据）需额外确认

### 4.2 依赖
- 任务执行引擎
- 通知模块
- 权限模块

## 5. 风险与问题

| ID | 风险/问题 | 影响 | Mitigation/解决方案 |
|----|-----------|------|---------------------|
| RISK-001 | 高危脚本误执行 | 高 | 权限控制和二次确认 |
| RISK-002 | 脚本执行超时 | 中 | 超时控制和中断机制 |

## 6. 验收标准

1. 脚本上传、编辑、删除功能正常
2. 版本历史可查看和回滚
3. 支持单次、定时、周期执行
4. 执行日志完整记录
5. 执行结果通知正常发送
