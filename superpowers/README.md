# ⚡ Superpowers - 执行层

> 管"代码写得对不对" —— 执行层的工程纪律

## 职责

- ✅ 代码质量检查 (Lint, Format, Type Check)
- ✅ 单元测试与集成测试
- ✅ CI/CD 流水线自动化
- ✅ 部署脚本与回滚机制
- ✅ 监控告警规则
- ✅ 自动巡检与健康检查

## 目录结构

```
superpowers/
├── ci/              # CI/CD 配置 (GitHub Actions, Jenkins, GitLab CI)
├── scripts/         # 部署、启动、停止等运维脚本
└── checks/          # 质量门禁规则
```

## 质量门禁

部署前必须通过以下检查：

1. **代码检查** - ESLint / Checkstyle / Pylint
2. **单元测试** - 测试覆盖率 ≥ 80%
3. **安全扫描** - 依赖漏洞检测
4. **构建验证** - 产物完整性校验
5. **环境一致性** - 配置与环境变量校验

## 流水线流程

```
代码提交 → 质量检查 → 单元测试 → 构建镜像 
        → 安全扫描 → 部署预发 → 冒烟测试 → 上线
```
